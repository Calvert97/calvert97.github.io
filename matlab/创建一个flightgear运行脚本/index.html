创建一个FlightGear运行脚本

发布 2021, May 05 by Calvert Lee 需要1分钟读完

要以所需的初始条件（位置，日期，时间，天气，操作模式）启动FlightGear，最好通过使用Generate Run Script Block或使用FlightGear随附的界面来创建运行脚本。

如果要为要链接到FlightGear的每个模型制作单独的运行脚本，然后将它们放置在单独的目录中，请在启动Simulink模型之前从MATLAB界面运行相应的脚本。

1. 使用Generate Run Script Block 创建运行脚本

  1. 打开飞行模拟器接口子库。

  2. 创建新的Simulink模型或打开现有模型。

  3. 将“生成运行脚本”块拖到Simulink图中。

  4. 双击“生成运行脚本”块。它的对话框打开。观察三个窗格，FlightGear，Network和File。

  5. 在“文件”选项卡的“输出文件名” 参数中，键入输出文件的名称。该名称应该是您要使用这些初始参数启动FlightGear的命令的名称。使用适当的文件扩展名：

    Win--*.bat Linux,Mac--*.sh

  6. 在“文件” 选项卡的FlightGear基本目录参数中 ，指定FlightGear安装文件夹的名称。

  7. 在“文件”选项卡的FlightGear几何体模型名称”参数中，在FlightGear/data/Aircraft文件夹中指定包含所需模型几何体的子文件夹的名称。

  8. 根据需要指定初始条件。

  9. 单击“参数”区域顶部的“生成脚本”按钮。 Aerospace Blockset软件将生成运行脚本，并将其保存在MATLAB工作文件夹中，该文件位于在“文件” >“输出文件名”字段中指定的文件名下。

  10. 选择或清除这些复选框，然后

  • 若要指示FlightGear在模拟器运行时自动安装所需的场景—选择 在模拟过程中安装FlightGear场景（需要Internet连接）。对于Windows系统，在启用此选项的情况下启动FlightGear时可能会遇到错误消息。有关更多信息，请参阅安装其他FlightGear风景。
  • 禁用FlightGear着色器选项—选择“ 禁用FlightGear着色器选项”。
  1. 如果需要，重复步骤5到10以生成其他运行脚本。
  2. 单击“确定”关闭对话框。您不需要使用Simulink模型保存“生成运行脚本”块。

2. 使用FlightGear自带的界面

FlightGear启动器GUI（FlightGear的一部分，而不是Aerospace Blockset产品的一部分）使您可以在可见的FlightGear运行命令中构建简单和高级选项。

  1. 在MATLAB命令窗口中，选择快捷方式，然后点击新建快捷方式。将打开“快捷方式编辑器”对话框。

  2. 设置Label，Callback，Category和Icon字段，如下图所示。

  3. 点击保存。该FlightGear的按钮会出现在你的桌面MATLAB。如果单击它，则输出文件（例如runfg.bat）将在当前文件夹中运行。

完成设置后，启动FlightGear并运行模型：

  1. 确保您的模型在可写文件夹中。打开模型，然后更新图。此步骤可确保在运行之前编译所有引用的块代码，并确保已编译框图。启动FlightGear后，它将在运行时使用所有可用的处理器电源。
  2. 单击FlightGear按钮或手动运行FlightGear运行脚本。
  3. 当FlightGear启动时，它将在运行脚本中指定的初始坐标处显示初始视图。如果您在不同的计算机上运行Simulink软件和FlightGear，请安排同时查看两个显示器。
  4. 现在开始模拟并在FlightGear中查看动画。
